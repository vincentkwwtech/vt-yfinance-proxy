# VT YFinance Proxy 系统使用手册

## 1. 快速入门

### 1.1 服务地址

- **API 基地址：** `http://{host}:{port}`（默认 `http://localhost:8000`）
- **Swagger 交互文档：** `http://localhost:8000/docs`
- **ReDoc 文档：** `http://localhost:8000/redoc`

### 1.2 健康检查

```bash
curl http://localhost:8000/health
```

返回：

```json
{"status": "ok"}
```

## 2. 金融数据 API 使用说明

所有金融数据 API 的基础路径为 `/finance-data/yfinance`，通过 `ticker` 查询参数指定股票代码。

### 2.1 获取公司概况

```bash
curl "http://localhost:8000/finance-data/yfinance/profile?ticker=AAPL"
```

### 2.7 获取 K线数据

**基础用法（默认 1 年日线）：**

```bash
curl "http://localhost:8000/finance-data/yfinance/kline?ticker=AAPL"
```

**自定义周期和间隔：**

```bash
# 3 个月的周线
curl "http://localhost:8000/finance-data/yfinance/kline?ticker=AAPL&period=3mo&interval=1wk"

# 5 天的 15 分钟线
curl "http://localhost:8000/finance-data/yfinance/kline?ticker=AAPL&period=5d&interval=15m"

# 1 天的 1 分钟线（含盘前盘后）
curl "http://localhost:8000/finance-data/yfinance/kline?ticker=AAPL&period=1d&interval=1m&prepost=true"
```

**可用参数：**

| 参数 | 可选值 |
|------|--------|
| period | `1d` `5d` `1mo` `3mo` `6mo` `1y` `2y` `5y` `10y` `ytd` `max` |
| interval | `1m` `2m` `5m` `15m` `30m` `60m` `90m` `1h` `1d` `5d` `1wk` `1mo` `3mo` |

> **注意：** 分钟级数据（1m~90m）的历史深度有限，通常仅支持最近 7-30 天。

**返回示例：**

```json
{
  "ticker": "AAPL",
  "period": "1mo",
  "interval": "1d",
  "prepost": false,
  "data": [
    {
      "Date": "2024-01-02",
      "Open": 185.5,
      "High": 186.2,
      "Low": 184.3,
      "Close": 185.8,
      "Volume": 45000000
    }
  ]
}
```

### 2.8 获取全量数据

一次性获取所有维度的数据：

```bash
curl "http://localhost:8000/finance-data/yfinance/all?ticker=AAPL"
```

返回包含 `profile`、`shareholders`、`actions`、`analyst`、`financial`、`optionchain`、`kline` 七个子对象的聚合数据。

> **注意：** 此接口调用耗时较长，建议仅在需要全量数据时使用。

## 5. 错误处理

### 5.1 错误码说明

| HTTP 状态码 | 含义 | 常见原因 |
|-------------|------|---------|
| 200 | 成功 | 请求正常处理 |
| 404 | 未找到 | 股票代码无效或不存在 |
| 422 | 参数错误 | 缺少必填参数 `ticker`，或参数格式错误 |
| 500 | 服务器内部错误 | 数据获取失败、参数值不合法（如无效 period/interval） |
| 502 | 网关错误 | yfinance 请求失败（Legacy API） |

### 5.2 错误返回格式

```json
{
  "detail": "Invalid ticker: XXXXX"
}
```

```json
{
  "detail": "Invalid period 'abc'. Valid: 1d, 5d, 1mo, ..."
}
```

## 6. 使用建议

### 6.1 性能建议

- 避免高频调用同一股票代码，Yahoo Finance 有频率限制
- 如需多个数据维度，优先使用 `/all` 端点减少请求次数
- 分钟级 K线数据量较大，建议按需获取

### 6.2 数据说明

- 数据来源为 Yahoo Finance，存在 15 分钟左右的延迟（非实时行情）
- 部分字段可能为 `null`，取决于 Yahoo Finance 的数据覆盖范围
- 不同市场的交易时间不同，非交易时段可能无法获取最新数据
- A股和港股数据的覆盖度可能不如美股完整

### 6.3 交互式文档

推荐通过浏览器访问 Swagger UI（`http://localhost:8000/docs`）进行交互式 API 测试：

1. 打开浏览器访问 `http://localhost:8000/docs`
2. 找到要测试的端点，点击展开
3. 点击 "Try it out" 按钮
4. 填入参数（如 ticker=AAPL）
5. 点击 "Execute" 查看返回结果

## 7. 接入示例

### 7.1 Python

```python
import requests

BASE_URL = "http://localhost:8000"

# 获取公司概况
resp = requests.get(f"{BASE_URL}/finance-data/yfinance/profile", params={"ticker": "AAPL"})
profile = resp.json()
print(f"公司: {profile['name']}, 市值: {profile['marketCap']}")

# 获取 K 线数据
resp = requests.get(f"{BASE_URL}/finance-data/yfinance/kline", params={
    "ticker": "AAPL",
    "period": "1mo",
    "interval": "1d"
})
kline = resp.json()
for bar in kline["data"]:
    print(f"{bar['Date']}: O={bar['Open']} H={bar['High']} L={bar['Low']} C={bar['Close']}")
```

### 7.2 JavaScript / Node.js

```javascript
const BASE_URL = "http://localhost:8000";

// 获取实时报价
const resp = await fetch(`${BASE_URL}/api/v1/stocks/AAPL/quote`);
const quote = await resp.json();
console.log(`AAPL 当前价格: ${quote.price}`);
```

### 7.3 curl 批量查询

```bash
# 批量获取多只股票概况
for ticker in AAPL MSFT GOOGL AMZN; do
  echo "=== $ticker ==="
  curl -s "http://localhost:8000/finance-data/yfinance/profile?ticker=$ticker" | python3 -m json.tool
done
```
