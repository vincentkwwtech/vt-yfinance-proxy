# VT YFinance Proxy 系统安装部署手册

## 1. 环境要求

### 1.1 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 1 核 | 2 核 |
| 内存 | 512 MB | 1 GB |
| 磁盘 | 500 MB | 1 GB |
| 网络 | 可访问 Yahoo Finance（直连或通过代理） | — |

### 1.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | >= 3.10 | 推荐 3.12 |
| pip | 最新版本 | Python 包管理器 |
| Docker（可选） | >= 20.0 | Docker 部署方式 |
| Git | 任意 | 获取源代码 |
| 操作系统 | Linux（推荐 Ubuntu/CentOS） | 也支持 macOS/Windows |

---

## 2. 获取源代码

```bash
git clone https://github.com/vkww-tech/vt-yfinance-proxy.git
cd vt-yfinance-proxy
```

---

## 3. 环境配置

### 3.1 配置环境变量

复制示例配置文件并根据需要修改：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 代理配置（可选，按需配置）
YFINANCE_PROXY=http://proxy.example.com:8080

# 服务监听地址
HOST=0.0.0.0

# 服务监听端口
PORT=8000

# 日志级别
LOG_LEVEL=INFO
```

**代理格式示例：**

| 代理类型 | 格式 |
|---------|------|
| HTTP | `http://proxy.example.com:8080` |
| HTTPS | `https://proxy.example.com:8443` |
| SOCKS5 | `socks5://proxy.example.com:1080` |
| 带认证 | `http://user:password@proxy.example.com:8080` |

> 如果服务器可以直接访问 Yahoo Finance，则无需配置 `YFINANCE_PROXY`。

## 4. 部署方式一：开发环境（本地运行）

### 4.1 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 4.2 安装依赖

```bash
pip install -r requirements.txt
pip install -e .
```

### 4.3 启动服务

```bash
python main.py
```

或使用 uvicorn 直接启动（支持热重载）：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 4.4 验证服务

```bash
curl http://localhost:8000/health
# 预期返回: {"status":"ok"}
```

## 5. 部署方式二：Docker 部署（推荐）

### 5.1 一键部署

```bash
chmod +x deploy-docker.sh
./deploy-docker.sh
```

脚本将自动完成：构建镜像 → 停止旧容器 → 启动新容器。

### 5.2 手动部署

#### 构建镜像

```bash
docker build -t vt-yfinance-proxy .
```

#### 启动容器

```bash
# 无代理
docker run -d \
  --name vt-yfinance-proxy \
  --restart unless-stopped \
  -p 8000:8000 \
  vt-yfinance-proxy

# 带代理
docker run -d \
  --name vt-yfinance-proxy \
  --restart unless-stopped \
  -p 8000:8000 \
  -e YFINANCE_PROXY=http://proxy.example.com:8080 \
  vt-yfinance-proxy
```

#### 自定义端口

```bash
docker run -d \
  --name vt-yfinance-proxy \
  --restart unless-stopped \
  -p 9090:8000 \
  -e HOST=0.0.0.0 \
  -e PORT=8000 \
  vt-yfinance-proxy
```

### 5.3 容器管理

```bash
# 查看运行状态
docker ps | grep vt-yfinance-proxy

# 查看日志
docker logs vt-yfinance-proxy
docker logs -f vt-yfinance-proxy    # 实时跟踪

# 停止服务
docker stop vt-yfinance-proxy

# 重启服务
docker restart vt-yfinance-proxy

# 删除容器
docker rm -f vt-yfinance-proxy
```

### 5.4 验证服务

```bash
curl http://localhost:8000/health
# 预期返回: {"status":"ok"}
```

## 6. 部署方式三：Systemd 部署（Linux 生产环境）

### 6.1 一键部署

```bash
chmod +x deploy-systemd.sh
sudo ./deploy-systemd.sh
```

脚本将自动完成：

1. 创建 Python 虚拟环境并安装依赖
2. 创建 systemd 服务单元文件
3. 配置日志轮转
4. 启动并设置开机自启

### 6.2 自定义配置

部署前可通过环境变量自定义：

```bash
# 自定义端口和代理
PORT=9090 YFINANCE_PROXY=http://proxy:8080 sudo -E ./deploy-systemd.sh
```

**可配置参数：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `HOST` | 0.0.0.0 | 监听地址 |
| `PORT` | 8000 | 监听端口 |
| `PYTHON_BIN` | python3 | Python 解释器路径 |
| `USER_NAME` | 当前用户 | 服务运行用户 |
| `GROUP_NAME` | 当前用户 | 服务运行用户组 |
| `VENV_DIR` | .venv | 虚拟环境目录 |
| `LOG_DIR` | /var/log/vt-yfinance-proxy | 日志目录 |
| `YFINANCE_PROXY` | 无 | 代理地址 |

### 6.3 服务管理

```bash
# 查看状态
sudo systemctl status vt-yfinance-proxy

# 启动服务
sudo systemctl start vt-yfinance-proxy

# 停止服务
sudo systemctl stop vt-yfinance-proxy

# 重启服务
sudo systemctl restart vt-yfinance-proxy

# 查看日志
sudo journalctl -u vt-yfinance-proxy -f

# 禁用/启用开机自启
sudo systemctl disable vt-yfinance-proxy
sudo systemctl enable vt-yfinance-proxy
```

### 6.4 日志管理

日志文件位于 `/var/log/vt-yfinance-proxy/app.log`，已配置 logrotate 自动轮转：

- 轮转周期：每周
- 保留数量：4 份
- 压缩：启用
- 空文件不轮转

### 6.5 卸载服务

```bash
sudo systemctl stop vt-yfinance-proxy
sudo systemctl disable vt-yfinance-proxy
sudo rm /etc/systemd/system/vt-yfinance-proxy.service
sudo rm /etc/logrotate.d/vt-yfinance-proxy
sudo systemctl daemon-reload
```

## 8. 常见问题排查

### 8.1 服务无法启动

**问题：** 端口被占用

```bash
# 检查端口占用
lsof -i :8000
# 修改 PORT 环境变量使用其他端口
```

**问题：** Python 版本不满足要求

```bash
python3 --version
# 确保版本 >= 3.10
```

### 8.2 无法获取数据

**问题：** 网络不通

```bash
# 测试 Yahoo Finance 连通性
curl -I https://finance.yahoo.com
```

**问题：** 代理配置错误

```bash
# 通过代理测试连通性
curl -x http://proxy:8080 -I https://finance.yahoo.com
```

### 8.3 Docker 相关

**问题：** 容器启动后立即退出

```bash
# 查看容器日志
docker logs vt-yfinance-proxy
```

**问题：** 镜像构建失败

```bash
# 清理 Docker 缓存重新构建
docker build --no-cache -t vt-yfinance-proxy .
```

### 8.4 数据返回 404

检查股票代码是否正确。常见代码格式：

| 市场 | 格式 | 示例 |
|------|------|------|
| 美股 | 直接代码 | AAPL, MSFT, GOOGL |
| 港股 | 数字.HK | 0700.HK, 9988.HK |
| A股 | 代码.SS（上海）/ 代码.SZ（深圳） | 600519.SS, 000001.SZ |
| 指数 | ^代码 | ^GSPC, ^VIX, ^HSI |
