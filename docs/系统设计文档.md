# VT YFinance Proxy 系统设计文档

## 1. 系统概述

### 1.1 项目背景

在内部网络环境中，直接访问 Yahoo Finance 获取金融数据存在网络限制。VT YFinance Proxy 作为代理服务，封装 yfinance Python 库，对外提供标准化的 RESTful API，解决内部系统获取金融数据的需求。

### 1.2 设计目标

- 提供统一的金融数据查询 API
- 支持通过代理服务器（HTTP/HTTPS/SOCKS5）访问 Yahoo Finance
- 轻量部署，支持 Docker 和 Systemd 两种运行方式
- 自动生成 API 文档，降低接入成本

### 1.3 技术选型

| 类别 | 技术 | 版本要求 | 选型理由 |
|------|------|---------|---------|
| Web 框架 | FastAPI | >=0.110 | 高性能异步框架，自动生成 OpenAPI 文档 |
| ASGI 服务器 | Uvicorn | >=0.27 | 轻量高效的 ASGI 服务器 |
| 数据源 | yfinance | >=0.2.36 | 成熟的 Yahoo Finance Python 封装库 |
| 数据处理 | pandas | >=2.2 | DataFrame 处理金融时序数据 |
| 数据校验 | Pydantic | >=2.6 | FastAPI 原生支持的数据模型校验 |
| 配置管理 | python-dotenv | >=1.0 | 环境变量管理 |
| 运行环境 | Python | >=3.10 | 类型提示与现代语法支持 |
| 容器化 | Docker | python:3.12-slim | 轻量生产镜像 |

## 3. 模块设计

### 3.1 应用入口（app/main.py）

- 创建 FastAPI 应用实例
- 注册路由：`/api/v1`（legacy）和 `/finance-data/yfinance`（v2）
- 配置日志级别（通过 `LOG_LEVEL` 环境变量）
- 提供 `/health` 健康检查端点

### 3.2 数据模型（app/schemas.py）

```
Candle
├── date: date           # 交易日期
├── open: float          # 开盘价
├── high: float          # 最高价
├── low: float           # 最低价
├── close: float         # 收盘价
└── volume: int          # 成交量

DailySeriesResponse
├── symbol: str          # 股票代码
└── candles: List[Candle]# K线数据列表

QuoteResponse
├── symbol: str          # 股票代码
├── price: float?        # 当前价格
├── previous_close: float?
├── open: float?
├── day_high: float?
├── day_low: float?
├── volume: int?
├── currency: str?
├── exchange: str?
└── market_time: datetime?
```

### 3.3 路由层设计

#### finance_data.py — 金融数据路由

- 统一参数：`ticker` 查询参数
- 统一异常处理：
  - `InvalidTickerError` → HTTP 404
  - `FinanceDataError` → HTTP 500
- 8 个端点：profile、shareholders、actions、analyst、financial、optionchain、kline、all

#### market.py — 行情路由（Legacy）

- 路径参数：`symbol`
- 日期过滤：可选 `start`、`end` 查询参数
- 3 个端点：stocks daily、vix daily、stock quote
- 异常处理：`YFinanceError` → HTTP 502

### 3.4 服务层设计

#### finance_data_service.py

**核心工具函数：**

| 函数 | 功能 |
|------|------|
| `_get_proxy()` | 读取 `YFINANCE_PROXY` 环境变量 |
| `_get_ticker(symbol)` | 创建带代理配置的 yfinance.Ticker 对象 |
| `_validate_ticker(ticker)` | 通过 info 字段验证代码有效性 |
| `_safe_float(val)` | 安全转换浮点数（处理 NaN） |
| `_safe_int(val)` | 安全转换整数 |
| `_safe_str(val)` | 安全转换字符串 |
| `_df_to_records(df)` | DataFrame 转字典列表（处理时间戳和 NaN） |

**业务函数：**

| 函数 | 功能 | 数据源 |
|------|------|--------|
| `get_profile(symbol)` | 公司概况 | ticker.info |
| `get_shareholders(symbol)` | 股东信息 | ticker.major_holders 等 |
| `get_actions(symbol)` | 公司行为 | ticker.actions / dividends / splits |
| `get_analyst(symbol)` | 分析师评级 | ticker.upgrades_downgrades 等 |
| `get_financial(symbol)` | 财务数据 | ticker.income_stmt 等 + info |
| `get_optionchain(symbol, exp)` | 期权链 | ticker.options / option_chain |
| `get_kline(symbol, ...)` | K线数据 | ticker.history() |
| `get_all(symbol)` | 全量聚合 | 以上所有函数 |

#### yfinance_service.py

| 函数 | 功能 |
|------|------|
| `get_daily_ohlcv(symbol, start, end)` | 日线 OHLCV 数据（返回 Candle 列表） |
| `get_latest_quote(symbol)` | 最新报价（多级降级策略：fast_info → info → history） |

## 4. 关键设计决策

### 4.1 同步 vs 异步

系统采用**同步**调用方式。yfinance 库本身为同步阻塞 IO，FastAPI 会自动将同步端点放入线程池执行，避免阻塞事件循环。

### 4.2 数据安全转换

由于 yfinance 返回的数据包含 pandas 的 `NaN`、`NaT` 等特殊值，这些值不能直接序列化为 JSON。系统通过 `_safe_*` 系列工具函数和 `_df_to_records` 进行统一转换：

- `NaN` / `None` → `None`（JSON null）
- `Timestamp` → ISO 格式字符串
- `DataFrame` → `List[Dict]`

### 4.3 Ticker 验证策略

通过访问 `ticker.info` 并检查 `shortName` 或 `longName` 是否存在来判断股票代码是否有效。无效代码抛出 `InvalidTickerError`，映射为 HTTP 404。

### 4.4 报价降级策略（Legacy 服务）

`get_latest_quote` 采用三级降级策略获取报价：

1. **首选**：`ticker.fast_info` — 速度最快
2. **降级1**：`ticker.info` — 字段更全
3. **降级2**：近5日历史收盘价 — 最后兜底

### 4.5 代理支持

代理配置通过 `YFINANCE_PROXY` 环境变量注入，在创建 `yfinance.Ticker` 对象时统一设置，支持 HTTP、HTTPS、SOCKS5 协议。

### 4.6 参数校验

K线端点（kline）在服务层校验 `period` 和 `interval` 参数的合法性，使用白名单集合匹配。不合法参数抛出 `FinanceDataError`。

## 6. 数据流

### 6.1 典型请求处理流程

```
Client Request
     │
     ▼
FastAPI Router          ← 解析参数、选择处理函数
     │
     ▼
Service Function        ← 创建 Ticker、校验、调用 yfinance
     │
     ▼
yfinance Library        ← 通过代理访问 Yahoo Finance
     │
     ▼
数据转换                 ← DataFrame → Dict，NaN → None
     │
     ▼
JSON Response           ← 返回标准化 JSON 数据
```

### 6.2 异常处理流程

```
Service 抛出异常
     │
     ├─ InvalidTickerError ──→ Router 捕获 ──→ HTTP 404
     ├─ FinanceDataError   ──→ Router 捕获 ──→ HTTP 500
     └─ YFinanceError      ──→ Router 捕获 ──→ HTTP 502
```

## 8. 限制与约束

| 约束 | 说明 |
|------|------|
| 无缓存 | 每次请求直接调用 yfinance，无本地缓存 |
| 无速率限制 | 系统未实现限流，依赖 Yahoo Finance 的频率限制 |
| 无 CORS 配置 | 默认不配置跨域，浏览器直接访问需额外配置 |
| 无认证 | 无鉴权机制，需通过网络层控制访问 |
| 同步调用 | 高并发场景下受线程池大小限制 |
| 数据时效性 | 依赖 Yahoo Finance 的数据更新频率 |
| 分钟级 K线限制 | yfinance 对分钟级数据的历史深度有限制（通常 7-30 天） |
